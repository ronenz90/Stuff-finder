# 🔎 צייד המציאות — מערכת מעקב מוצרים חינמית

מערכת שסורקת את הרשת לפי לוח זמנים, מחפשת מכירות של מוצרים שהגדרת (בכמה שפות),
**מאמתת שהמודעה חיה ולא נמכרה**, ושולחת התראה במייל עם קישור ישיר.

**עלות: 0 ₪.** הכל רץ על שירותים חינמיים: GitHub Pages, GitHub Actions, DuckDuckGo, Gmail.

## התקנה — 6 שלבים, ~10 דקות

### שלב 1: יצירת ריפו

1. היכנס ל-GitHub ← **New repository**
2. שם: `product-tracker` (או כל שם), סוג: **Public** (נדרש ל-Actions חינמי ללא הגבלה ול-Pages)
3. העלה את כל קבצי הפרויקט הזה לריפו (Add file ← Upload files), כולל התיקייה `.github/workflows`

> ⚠️ אם `scan.yml` לא עולה דרך הדפדפן בתוך תיקייה — צור אותו ידנית:
> Add file ← Create new file ← בשם הקובץ הקלד `.github/workflows/scan.yml` והדבק את התוכן.

### שלב 2: הפעלת GitHub Pages

Settings ← Pages ← Branch: **main**, תיקייה: **/ (root)** ← Save.
תוך דקה-שתיים לוח הבקרה יהיה זמין ב:
`https://YOUR-USERNAME.github.io/product-tracker/`

### שלב 3: שירות מייל חינמי (בלי אימות דו-שלבי)

מומלץ: **Brevo** — 300 מיילים ביום בחינם, הרשמה של 2 דקות, בלי צורך בדומיין:

1. הירשם בחינם: https://www.brevo.com
2. אחרי האימות: תפריט עליון ← השם שלך ← **SMTP & API** ← לשונית **SMTP**
3. לחץ **Generate a new SMTP key** והעתק את הפרטים:
   - Server: `smtp-relay.brevo.com` · Port: `587`
   - Login: המייל שנרשמת איתו · Password: מפתח ה-SMTP שנוצר

חלופות חינמיות (אותם 5 Secrets, ערכים אחרים):

| שירות | מכסה חינמית | שרת SMTP | פורט |
|---|---|---|---|
| **Brevo** ⭐ | 300/יום | smtp-relay.brevo.com | 587 |
| Mailjet | 200/יום | in-v3.mailjet.com | 587 |
| SendGrid | 100/יום | smtp.sendgrid.net (user: `apikey`) | 587 |
| Gmail | 500/יום | smtp.gmail.com (דורש 2FA + סיסמת אפליקציה) | 465 |

### שלב 4:### שלב 4: הגדרת Secrets בריפו

Settings ← Secrets and variables ← **Actions** ← New repository secret. צור שלושה:

| שם | ערך (דוגמה ל-Brevo) |
|---|---|
| `SMTP_HOST` | `smtp-relay.brevo.com` |
| `SMTP_PORT` | `587` |
| `SMTP_USER` | המייל שנרשמת איתו ל-Brevo |
| `SMTP_PASSWORD` | מפתח ה-SMTP משלב 3 |
| `SMTP_FROM` | כתובת השולח שתופיע במייל (בד"כ זהה ל-SMTP_USER) |
| `ALERT_EMAIL` | כתובת ברירת המחדל **לקבלת** ההתראות |

> מי שכבר הגדיר `GMAIL_USER`/`GMAIL_APP_PASSWORD` — ימשיך לעבוד כרגיל (תאימות לאחור).

#### 📧 מייל שונה לכל מוצר — מוצפן אוטומטית
בטופס הוספת המוצר יש שדה "מייל להתראות על מוצר זה". הכתובת **לא נשמרת בקבצים הציבוריים**:

1. הממשק גוזר מהכתובת שם Secret קבוע (למשל `ALERT_EMAIL_A1B2C3D4E5F6`)
2. בודק אם Secret כזה כבר קיים בריפו — אם כן, משתמש בו (אותה כתובת = אותו Secret, בלי כפילויות)
3. אם לא — מצפין את הכתובת עם המפתח הציבורי של הריפו (libsodium sealed box) ויוצר Secret חדש
4. ב-`products.json` נשמרים רק שם המפתח ורמז מוסווה לתצוגה (`co***@gm***`)
5. הסורק מקבל את כל ה-Secrets בזמן ריצה ומפענח את הכתובת **בזיכרון בלבד**, ברגע השליחה

מוצר ללא כתובת משתמש ב-`ALERT_EMAIL` הכללי. כתובת המשמשת כמה מוצרים מקבלת מייל אחד מרוכז.

### שלב 5: טוקן ללוח הבקרה

1. GitHub ← Settings (של המשתמש) ← Developer settings ← **Fine-grained personal access tokens** ← Generate new token
2. Repository access: **Only select repositories** ← בחר את `product-tracker`
3. Permissions ← Repository permissions:
   - **Contents: Read and write** (שמירת מוצרים ותוצאות)
   - **Actions: Read and write** (כפתור "הרצת סריקה עכשיו")
   - **Workflows: Read and write** (שינוי תדירות הסריקה מהממשק)
   - **Secrets: Read and write** (הצפנת כתובות מייל פר-מוצר)
4. העתק את הטוקן, פתח את לוח הבקרה (הכתובת משלב 2) ← לשונית **הגדרות** ← הדבק והתחבר.

הטוקן נשמר **רק בדפדפן שלך** (localStorage) ולא נשלח לשום מקום מלבד GitHub.

### שלב 6: בדיקה ראשונה

1. בלוח הבקרה הוסף מוצר (או השתמש בדוגמה הקיימת — גביע קידוש עירקי)
2. לחץ **▶ הרצת סריקה עכשיו** (או בריפו: Actions ← Product Scan ← Run workflow)
3. עקוב אחרי הריצה בלשונית Actions. בסיום — בדוק את המייל ואת לשונית 🔔 מציאות.

## ספקי חיפוש (כולם אופציונליים, כולם חינמיים)

המערכת עובדת מהקופסה עם DuckDuckGo בלבד. כל מפתח שתוסיף כ-Secret **משפר** את הכיסוי —
הסורק מזהה אוטומטית אילו מפתחות קיימים ומשתמש בכולם במקביל:

| ספק | מכסה חינמית | Secrets | יתרון |
|---|---|---|---|
| DuckDuckGo | ללא הגבלה רשמית | — (מובנה) | אפס הגדרות |
| **eBay Browse API** | 5,000 קריאות/יום | `EBAY_CLIENT_ID`, `EBAY_CLIENT_SECRET` | ⭐ מחזיר **רק מודעות פעילות** + מחיר — בלי סינון "נמכר" בכלל |
| Google Programmable Search | 100 קריאות/יום | `GOOGLE_CSE_KEY`, `GOOGLE_CSE_ID` | איכות החיפוש של גוגל |
| Brave Search API | 2,000 קריאות/חודש | `BRAVE_API_KEY` | אינדקס עצמאי, טוב לאתרים קטנים |

### הרשמה ל-eBay (מומלץ ביותר)
1. פתח חשבון מפתח חינמי: https://developer.ebay.com ← Join
2. My Account ← **Application Keys** ← בסביבת **Production** צור Keyset
3. העתק את **App ID (Client ID)** ואת **Cert ID (Client Secret)** ← הוסף כ-Secrets בריפו

### הרשמה ל-Google Programmable Search
1. צור מנוע חיפוש: https://programmablesearchengine.google.com ← Add ← "Search the entire web" ← העתק את **Search engine ID** (`GOOGLE_CSE_ID`)
2. צור מפתח API: https://developers.google.com/custom-search/v1/introduction ← Get a Key (`GOOGLE_CSE_KEY`)

### הרשמה ל-Brave
https://brave.com/search/api ← תוכנית **Free** ← העתק את המפתח (`BRAVE_API_KEY`)

## 📷 הוספת מוצר מתמונה בלבד

בלשונית 📦 מוצרים יש כרטיס "ניתוח תמונה ב-AI":

1. העתק את הפרומפט המובנה ופתח את ה-AI החינמי שלך **עם תמונת המוצר מצורפת**
2. **רקע עמוס?** הוסף שורה אחרי הפרומפט שמכוונת את ה-AI, למשל:
   `Focus on the silver cup in the center, ignore the tray behind it`
3. הפרומפט מנחה את ה-AI להתעלם מרקע/ידיים/חפצים אחרים, לתאר רק מה שנראה בפועל,
   ולא להמציא יוצר או תקופה (השערות מסומנות כ-"likely")
4. הדבק חזרה את ה-JSON — הטופס יתמלא אוטומטית: שם בעברית, שם באנגלית,
   תיאור חזותי ו-3-6 שאילתות חיפוש מוכנות

## איך עובד הסינון (ולמה אין התראות שווא)

כל תוצאת חיפוש עוברת משפך:

1. **ניקוד מילים** — תוצאות שלא מזכירות את שם המוצר נזרקות מיד
2. **אימות דף חי** — המערכת נכנסת לדף עצמו:
   - HTTP 404/410? ← נפסל (הלקח מהסימולציה 😉)
   - מכיל "Sold" / "נמכר" / "אזל מהמלאי" / "המכירה הסתיימה"? ← נפסל
   - יש מחיר / כפתור קנייה? ← מסומן כדף מכירה פעיל
3. **מניעת כפילויות** — כל URL שכבר דווח או נפסל נשמר ב-`found.json` ולא ייבדק שוב
4. **הכרעה**:
   - ציון ≥ 60% + דף מכירה פעיל ← **מייל מיידי** 📧
   - ציון גבולי ← נכנס ל-`pending_review.json` ← **סינון AI ידני** בלוח הבקרה

## סינון ה-AI (חינמי לגמרי)

אין צורך במפתח API בתשלום. בלשונית 🤖 בלוח הבקרה:

1. המערכת בונה **פרומפט מקצועי** הכולל את המוצרים, התיאורים החזותיים והתוצאות הגבוליות
2. לוחצים העתקה ← מדביקים ב-Claude / ChatGPT / Gemini (הגרסה החינמית מספיקה)
3. מדביקים חזרה את פלט ה-JSON ← המערכת מסננת, מציגה רק התאמות אמיתיות ושומרת אותן
