# 🔎 צייד המציאות — מערכת מעקב מוצרים חינמית

מערכת שסורקת את הרשת לפי לוח זמנים, מחפשת מכירות של מוצרים שהגדרת (בכמה שפות),
**מאמתת שהמודעה חיה ולא נמכרה**, ושולחת התראה במייל עם קישור ישיר.

**עלות: 0 ₪.** הכל רץ על שירותים חינמיים: GitHub Pages, GitHub Actions, DuckDuckGo, Gmail.

---

## מה המערכת יודעת לעשות

- **מעקב רב-לשוני** — שם המוצר בעברית, אנגלית ועוד, עם מילות חיפוש נוספות
- **הוספה מתמונה** — מנתחת תמונת מוצר ב-AI וממלאת את הטופס אוטומטית
- **אימות זמינות** — נכנסת לכל דף ופוסלת מה שנמכר / אזל / הסתיים / 404
- **סינון חכם** — מייל אוטומטי רק על התאמה מלאה + דף מכירה פעיל; השאר לסינון AI
- **תדירות פר-מוצר** — שעתי / יומי / שבועי / חודשי, לכל מוצר בנפרד
- **סינון מחיר** — תקרת מחיר פר-מוצר עם המרת מטבע אוטומטית
- **מייל פר-מוצר** — כל מוצר יכול להישלח לכתובת אחרת
- **התראות במייל** — עם קישור ישיר לדף המוצר

## התקנה — 6 שלבים

### שלב 1: יצירת ריפו
1. GitHub ← **New repository** ← שם כרצונך ← סוג **Public** (נדרש ל-Actions ו-Pages חינמיים)
2. העלה את כל קבצי הפרויקט, כולל `.github/workflows/scan.yml`
   > אם הקובץ בתיקייה לא עולה דרך הדפדפן: Add file ← Create new file ← הקלד `.github/workflows/scan.yml` והדבק את התוכן.

### שלב 2: הפעלת GitHub Pages
Settings ← Pages ← Branch: **main**, תיקייה: **/ (root)** ← Save.
לוח הבקרה יהיה זמין ב-`https://YOUR-USERNAME.github.io/REPO-NAME/`

### שלב 3: שליחת מייל דרך Gmail
זו הדרך היחידה החינמית שמאפשרת שליחה **לכל נמען, בלי דומיין**.
1. הפעל אימות דו-שלבי: https://myaccount.google.com/security ← "אימות דו-שלבי"
2. צור סיסמת אפליקציה: https://myaccount.google.com/apppasswords ← תן שם ← קבל 16 תווים
   (אפשר להעתיק עם הרווחים — הקוד מנקה אותם)

### שלב 4: הגדרת Secrets בריפו
Settings ← Secrets and variables ← **Actions** ← New repository secret:

| שם | ערך |
|---|---|
| `GMAIL_USER` | כתובת ה-Gmail שלך (השולחת) |
| `GMAIL_APP_PASSWORD` | 16 התווים משלב 3 |
| `ALERT_EMAIL` | כתובת ברירת המחדל **לקבלת** ההתראות |

> **מייל שונה לכל מוצר:** בטופס ההוספה יש שדה "מייל להתראות". הכתובת מוצפנת ונשמרת
> כ-Secret אוטומטית (לא בקובץ הציבורי). מוצר ללא כתובת ישתמש ב-`ALERT_EMAIL`.
> Gmail מאפשר שליחה לכל נמען, כך שהתראות פר-מוצר עובדות במלואן.

### שלב 5: טוקן ללוח הבקרה
1. GitHub ← Settings (של המשתמש) ← Developer settings ← **Fine-grained personal access tokens** ← Generate
2. Repository access: **Only select repositories** ← בחר את הריפו
3. Permissions ← Repository permissions:
   - **Contents: Read and write** (שמירת מוצרים ותוצאות)
   - **Actions: Read and write** (הרצת סריקה, שליחת התראות)
   - **Secrets: Read and write** (הצפנת כתובות מייל פר-מוצר)
4. העתק את הטוקן ← פתח את לוח הבקרה ← לשונית **הגדרות** (⚙) ← הדבק והתחבר.

הטוקן נשמר **רק בדפדפן שלך** ולא נשלח לשום מקום מלבד GitHub.

### שלב 6: בדיקה
1. Actions ← Product Scan ← Run workflow ← בשדה `test_email` כתוב `true` ← Run
2. המתן לסיום ← בדוק שהמייל הגיע (גם בספאם; סמן "לא ספאם" בפעם הראשונה)
3. אם עובד — הוסף מוצר בלוח הבקרה ולחץ **▶ הרצת סריקה עכשיו**

## תדירות הסריקה — פר-מוצר

לכל מוצר תדירות משלו: **שעתי / יומי / שבועי / חודשי** — נבחרת בטופס ההוספה
וניתנת לעריכה בכל רגע מרשימת המוצרים.

ה-workflow רץ קבוע כל שעה, והסורק בודק לכל מוצר מתי נסרק לאחרונה
(נשמר ב-`found.json`) ומדלג על מי שטרם הגיע זמנו. ריצות ביניים מסתיימות בשניות.
לחיצה על **▶ הרצת סריקה עכשיו** סורקת את *כל* המוצרים מיידית.

> תזמוני cron של GitHub אינם מדויקים לדקה ולעיתים מתעכבים — זה נורמלי.

## סינון לפי מחיר מקסימלי

לכל מוצר אפשר להגדיר תקרת מחיר + מטבע ($ / ₪ / € / £) בטופס ההוספה.
הסורק שולף שער יציג יומי (API חינמי ללא מפתח), ממיר את מחיר המודעה ל-USD,
ומשווה לתקרה. מודעה יקרה מהתקרה עוברת לתור סינון ה-AI במקום להתראה אוטומטית.

- שדה ריק = ללא הגבלת מחיר
- ההמרה דו-כיוונית: תקרה בשקלים תופסת מודעה בדולרים ולהפך
- מודעה בלי מחיר ברור (מספר בלי סמל מטבע) לא נפסלת — עוברת לסינון

## ספקי חיפוש (כולם אופציונליים, כולם חינמיים)

המערכת עובדת מהקופסה עם **DuckDuckGo בלבד** — בלי שום הגדרה.
כל מפתח שתוסיף כ-Secret **משפר** את הכיסוי — הסורק מזהה אוטומטית אילו מפתחות
קיימים ומשתמש בכולם במקביל:

| ספק | מכסה חינמית | Secrets להוסיף | יתרון |
|---|---|---|---|
| **DuckDuckGo** | ללא הגבלה רשמית | — (מובנה) | אפס הגדרות |
| **eBay Browse API** | 5,000 קריאות/יום | `EBAY_CLIENT_ID`, `EBAY_CLIENT_SECRET` | ⭐ מחזיר **רק מודעות פעילות** + מחיר — בלי סינון "נמכר" בכלל |
| **Google Programmable Search** | 100 קריאות/יום | `GOOGLE_CSE_KEY`, `GOOGLE_CSE_ID` | איכות החיפוש של גוגל |
| **Brave Search API** | 2,000 קריאות/חודש | `BRAVE_API_KEY` | אינדקס עצמאי, טוב לאתרים קטנים |

### הרשמה ל-eBay (מומלץ ביותר)
1. חשבון מפתח חינמי: https://developer.ebay.com ← Join
2. My Account ← **Application Keys** ← בסביבת **Production** צור Keyset
3. העתק את **App ID (Client ID)** ואת **Cert ID (Client Secret)** ← הוסף כ-Secrets

### הרשמה ל-Google Programmable Search
1. מנוע חיפוש: https://programmablesearchengine.google.com ← Add ← "Search the entire web" ← העתק **Search engine ID** (`GOOGLE_CSE_ID`)
2. מפתח API: https://developers.google.com/custom-search/v1/introduction ← Get a Key (`GOOGLE_CSE_KEY`)

### הרשמה ל-Brave
https://brave.com/search/api ← תוכנית **Free** ← העתק את המפתח (`BRAVE_API_KEY`)

## 📷 הוספת מוצר מתמונה

בטופס ההוספה, תפריט "יש רק תמונה? ניתוח ב-AI":
1. העתק את הפרומפט ← פתח AI חינמי (Claude / ChatGPT / Gemini) **עם התמונה מצורפת**
2. רקע עמוס? הוסף שורה שמכוונת את ה-AI: `Focus on the silver cup, ignore the tray`
3. הדבק חזרה את ה-JSON — הטופס מתמלא: שם עברי/אנגלי, תיאור, ומילות חיפוש

הפרומפט מנחה את ה-AI לזהות את הפריט ספציפית (סוג, מסורת, תקופה) עם רמת ביטחון,
במקום תיאור גנרי שלא מוצא כלום בחיפוש.

## שיטת מייל חלופית (Resend)

אם תרצה בעתיד **כתובת שולח מקצועית בלי חשיפת ה-Gmail** — אפשר לעבור ל-Resend.
זה דורש **דומיין משלך** (~10-40 ₪/שנה) שמאמתים ב-resend.com. אז השולח יהיה
`alerts@your-domain.com`. הגדר `RESEND_API_KEY` ו-`RESEND_FROM` והסורק יעדיף
Gmail אם הוא מוגדר, אחרת Resend. בלי דומיין, Gmail הוא הפתרון הטוב ביותר.
